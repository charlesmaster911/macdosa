## 1. 현재 고객 여정 흐름 매핑

```
[인지] → [관심] → [고려] → [행동] → [재방문]
  ↓         ↓        ↓        ↓         ↓
검색/SNS   랜딩     데모/   결제모달    없음
유입       도착    소셜프루프  →결제     (사실상
                             실패率↑    이탈)
```

### 각 단계 현재 상태

| 단계 | 현재 터치포인트 | 상태 |
|------|----------------|------|
| 인지 | 검색, SNS 추정 (OG 태그 미확인) | 미구축 |
| 관심 | Hero 섹션 + URL 입력창 | 구축됨 |
| 고려 | Demo 섹션, 소셜 프루프, How 섹션, 가격 섹션 | 구축됨 |
| 행동(Free) | URL 입력 → result.html 분석 | 구축됨 |
| 행동(유료) | 블러 오버레이 → 결제 모달 → 이메일 수집 | 베타(미완성) |
| 재방문 | 없음 | 완전 부재 |

## 2. GAP 분석 — 단계별 이탈 구간 및 누락 요소

### GAP-01: 인지 단계 — 유입 경로 전략 없음
**현상:** GA4 비활성화(`G-XXXXXXXXXX` 미교체). OG 이미지 없음(`og-image.png` 참조되나 파일 없음). SEO 메타 태그 title 하나만 존재.

**이탈 지점:** 검색 유입 불가 상태. 현재 184명 베타 수치가 있으나 유입 채널 불명.

**누락 항목:**
- og:title, og:description, og:image (SNS 공유 시 빈 카드)
- meta description (SEO)
- 카카오 공유 SDK 앱키 (`YOUR_KAKAO_APP_KEY` 미입력)
- GA4 미활성화 → 데이터 수집 불가

---

### GAP-02: 관심 단계 — Hero 메시지 분산
**현상:** Hero에 3개의 메시지가 경쟁 중.
1. "맥도사" 브랜드 네임
2. URL 입력창 (행동 유도)
3. 소셜 프루프 (신뢰 구축)
4. 캐릭터 배지 (브랜딩)

**이탈 지점:** 첫 화면에서 '맥(脈) 분석 시작' CTA 이전에 스크롤 없이 보이는 콘텐츠가 너무 많음. URL 입력창 바로 아래 28px 여백 후 소셜 프루프, 그 아래 캐릭터 배지 순서로 시선이 분산됨.

**누락 항목:**
- URL 입력창 아래 즉각적인 가치 증명 (예: "지금 맥북 URL 붙여넣으면 3분 안에 얼마 아낄 수 있는지 보여드립니다")
- 모바일에서 nav-links 숨겨짐(미디어쿼리) → "지금 분석하기" nav-btn만 남음. 이 버튼이 영웅적 역할.

---

### GAP-03: 고려 단계 — 데모가 설득력 충분하나 연결고리 끊김
**현상:** Demo 섹션에서 "813,310원 절약" 사례를 보여주고 블러로 막음 → 잠금 해제 CTA `9,900원으로 잠금 해제` 노출. 그런데 이 CTA를 눌렀을 때 실제로 결제 완료 시 콘텐츠를 받을 수 있는지 명확하지 않음 (버튼 텍스트: "베타 신청하기", 결제 아님).

**이탈 지점:**
- 랜딩의 데모 블러 CTA는 직접 결제처럼 보이나 실제론 이메일 수집 (베타)
- 결제 모달에 "정식 출시 후 토스페이먼츠 연동 예정 · 베타는 무료" 문구가 하단에 작게 표시 → 사용자가 결제가 실제로 이루어지는지 혼란
- 소셜 프루프 숫자(184명, 18만원, 3분) 출처 미명시

**누락 항목:**
- "지금 분석하면 바로 보임" vs "베타라 나중에 봄" 구분 명확화
- 데모 → 직접 URL 입력 연결 CTA ("내 맥북도 분석해보기 →")
- How 섹션(천지인) 이후 pricing 전 중간 CTA 없음

---

### GAP-04: 행동 단계 — 분석 결과 페이지에서 전환 마찰 과다
**현상:** result.html에 블러 오버레이가 4곳(구매 경로 2-3위, 카드 조합, 부가세 계산기, 1위 구매링크).

**이탈 지점:**
- 블러가 너무 많음 → 불신감. "이 서비스가 실제로 뭔가 가지고 있나?" 의문 유발
- 1위 구매 경로 요약(롯데하이마트, 1,450,627원)은 FREE로 보임 → 그런데 실제 구매링크는 잠김 → 정보는 주고 링크는 막는 어중간한 설계
- 결제 모달에 이메일 입력 필드만 있고 실제 결제 UI가 없음 (토스페이먼츠 미연동) → 사용자가 결제를 완료했다는 신호 없음
- 결제 후 reload로 tier 체크하는 설계지만, 실제 token 발급 없이 localStorage만으로 처리 → 새로고침 시 상태 초기화 가능

**누락 항목:**
- 결제 전 "이 분석은 실제로 동작합니다" 신뢰 신호 (예: 오늘 분석 완료 건수 실시간 표시)
- 결제 완료 후 명확한 컨펌 화면 (현재: 토스트 메시지 1.8초 후 사라짐)
- 분석 URL 저장 → 재방문 시 결과 복원 기능 없음

---

### GAP-05: 재방문 단계 — 완전 부재
**현상:** 결제 후 이메일 수집하나, 이 이메일로 어떤 후속 액션도 없음. 가격 하락 알림(Pro 기능)은 기획 단계. 분석 결과 북마크/저장 기능 없음.

**이탈 지점:** 일회성 서비스로 인식. 맥북을 이미 산 사람은 재방문 이유 없음.

**누락 항목:**
- 이메일 → "다른 맥북 가격 분석" 리마인더 (친구 추천, 가족 구매 등)
- 구매 완료 후 액세서리 추천 등 Post-purchase 콘텐츠
- Pro 전환 유도 시퀀스 (One-time 결제자 → Pro 업셀)
- 카카오 알림 옵트인 (가격 하락 알림)

## 3. 개선 제안 — 퍼널 최적화

### 3-1. 인지 & 유입

**[HIGH] SEO + OG 태그 즉시 삽입**
```html
<!-- index.html <head> 내 추가 -->
<meta name="description" content="맥북 URL만 넣으면 30일 시세·최저가·구매 타이밍을 3분 안에 알려드립니다. 184명이 평균 18만원 절약했습니다.">
<meta property="og:title" content="맥도사 — 맥북 최저가 타이밍, 도사처럼 알려드립니다">
<meta property="og:description" content="교육할인보다 더 싼 루트가 있습니다. URL 하나로 확인하세요.">
<meta property="og:image" content="https://macdosa.kr/og-image.png">
<meta property="og:url" content="https://macdosa.kr">
<meta name="twitter:card" content="summary_large_image">
```
효과: SNS 공유 시 카드 노출, 검색 클릭률(CTR) +20~40% 기대.

**[HIGH] GA4 즉시 활성화**
G-XXXXXXXXXX를 실제 측정 ID로 교체, 주석 해제. 지금 데이터 없이는 어떤 퍼널 최적화도 근거 없음.

**[MED] result.html 공유 기능 활성화 (바이럴 루프)**
카카오 공유 SDK 앱키 발급 후 `YOUR_KAKAO_APP_KEY` 교체. 현재 fallback으로 클립보드 복사만 작동. 분석 결과 페이지가 바이럴의 핵심 채널이 될 수 있음.

결과 공유 메시지 예시:
```
"맥북 Pro 14인치 M4, 교육할인보다 81만원 더 싸게 샀어요.
URL 하나만 넣으면 됩니다 → macdosa.kr/result?url=..."
```

**[LOW] 네이버 블로그/카페 타겟 콘텐츠**
"맥북 살 때 교육할인이 최저가가 아닌 이유" 유형의 SEO 콘텐츠. 구매 의사결정 단계에서 검색하는 사용자 포착.

### 3-2. 관심 단계 CTA 개선

**[HIGH] Hero URL 입력창 아래 Value Proposition 단순화**
현재: "맥도사는 Mac의 맥(脈)을 읽습니다. 30일 시세 흐름과 현재 딜을 3단계로 분석해 당신이 살 타이밍을 정확히 짚어드립니다."

교체 후:
```
URL 붙여넣기 → 3분 → 얼마나 아낄 수 있는지 바로 보임
무료 · 회원가입 없음
```
이유: 사용자는 지금 '맥(脈)'이 무엇인지에 관심 없음. "나 얼마 아낄 수 있어?" 가 유일한 관심사.

**[MED] 소셜 프루프 위치 재조정**
현재: URL 입력창 → 소셜 프루프 → 캐릭터 배지 순서.

변경: URL 입력창 → **"오늘 XX명이 분석 중"(실시간 또는 누적 카운터)** → 첫 CTA 완료.

소셜 프루프(184명, 18만원) 섹션은 Hero 아래 별도 섹션으로 이동. Hero는 입력창 하나에 집중.

**[MED] 데모 섹션에 "내 맥북도 분석하기" CTA 추가**
Demo 카드 하단 블러 아래, 또는 Demo 섹션 마지막에:

  </p>

  </button>

### 3-3. 전환(결제) 마찰 제거

**[HIGH] 베타 vs 실결제 메시지 통일**
현재 랜딩의 결제 모달: "베타 신청하기" 버튼, 하단 "베타는 무료"  
현재 result.html 결제 모달: "결제하고 잠금 해제" 버튼

두 페이지의 결제 흐름이 다름 → 사용자 혼란. 베타 기간이면 명확히 "베타: 이메일만 입력하면 전체 결과 무료 공개"로 통일. 결제 연동 완료 후 즉시 "9,900원 결제" 흐름으로 전환.

**[HIGH] 블러 구역 축소 — 최소 1개 실질 데이터 공개**
현재: 2-3위 경로, 카드 조합, 부가세 계산기, 구매 링크 전부 블러.

개선안: 2위 경로 스토어명(가격은 블러), 카드사 이름(할인율 블러) 공개.
이유: 사용자가 "진짜 정보가 있다"고 믿어야 결제함. 전부 막으면 "가짜 아냐?" 의심 유발.

```
[공개] 2위: 쿠팡 로켓배송 ██████원 (구체적 경로 잠김)
[공개] 3위: 네이버 ██████원 (구체적 경로 잠김)
```

**[HIGH] 결제 완료 후 즉각 가치 제공 (토스트 개선)**
현재: 토스트 메시지 1.8초 → 사라짐. 블러 해제.

개선: 결제/이메일 입력 완료 직후 페이지 내 "잠금 해제됨" 애니메이션 + 자동 스크롤 to 구매 경로 섹션. 사용자가 바로 가치를 눈으로 확인해야 재결제/Pro 전환 의향 생김.

**[MED] 부가세 계산기 — FREE 부분 공개 전략**
현재: 전체 블러.

개선: "정가 → 최적경로 → 사업자 실질 부담" 3단계 흐름 요약은 FREE 공개 (현재 `vatSection` 상단에 이미 구현됨). 사업자번호 입력·정확한 수치만 잠금.
이미 구현된 구조를 활용 → 추가 개발 없이 즉시 적용 가능.

**[MED] result.html — "오늘만 유효" 긴박감 추가**
맥도사 판단 박스(verdictBox) 아래:
```html
<div style="text-align:center;font-size:12px;color:#FF9500;margin-top:-20px;margin-bottom:28px">
  ⏱ 이 딜은 오늘 자정까지만 유효합니다
</div>
```
원데이딜 특성상 실제로 긴박감이 사실에 근거함. 전환율 상승 효과.

### 3-4. 재방문 및 LTV 확장

**[HIGH] 결제/이메일 수집 후 자동 이메일 시퀀스**
- D+0: "분석 결과 PDF 요약" 이메일 (결과 영구 보관 목적)
- D+3: "이번 주 맥북 원데이딜 예고" (Pro 가치 체험)
- D+7: "친구에게 공유하면 분석 1회 무료 추가" (바이럴)
- D+14: Pro 전환 제안 ("가격 하락 알림 받기 → 19,900원/월")

구현: Resend API (FEATURE_SPEC에 이미 선택됨) + 이메일 템플릿 3개.

**[MED] "분석 저장" 기능 — URL 기반 결과 링크**
현재: result.html은 URL 파라미터로 분석하나 결과는 휘발성.

개선: 분석 ID를 URL에 포함 (`/result?id=abc123`)하여 결과 영구 저장 및 공유 가능. Supabase analyses 테이블에 이미 설계됨.
- 분석 완료 시 "이 결과 저장하기" CTA → 이메일 입력
- 공유 URL로 친구에게 전송 → 유입 채널 확장

**[MED] Pro 구독 업셀 — One-time 결제자 타겟**
One-time 결제 완료 7일 후 이메일:
```
"지난주 분석한 맥북 Pro 14인치, 지금 가격 어떻게 됐는지 아세요?
Pro 구독하면 실시간으로 알려드립니다 — 월 19,900원"
```

**[LOW] 중고 시장 섹션 활용 — 리텐션 콘텐츠**
result.html에 중고 시장 섹션(usedMarketSection) 이미 설계됨. 구매 완료 후 "팔 때는 얼마?" 정보로 재방문 유도 가능.

## 4. 글로벌 확장 고객 여정 변화 고려사항

### 핵심 전제
맥도사의 가치: 한국 특수 채널(롯데하이마트 원데이딜, 교육할인, 한국 카드사 즉시할인, 부가세 환급) 조합. 이 가치는 **한국 시장에만 완전히 작동**.

### 글로벌 확장 시 여정 변화

**인지 단계 변화**
- 언어: 영어 랜딩 별도 필요 (UI 복사 일부 한자/한글 혼용)
- 채널: 레딧(r/macbookpro, r/apple), 트위터/X, 유튜브 리뷰어 협업
- SEO 키워드: "MacBook cheapest price", "MacBook price tracker", "best time to buy MacBook"

**고려 단계 변화**
- 소셜 프루프 변경: "184명 한국인"은 해외 설득력 낮음. 누적 절약액("$500,000 saved") 방식 전환
- 신뢰 기제: 한국 카드사 조합은 무의미 → 신용카드 캐시백(Chase, Amex, Discover), 쿠폰 스택킹(미국), GST 환급(호주), VAT 환급(영국/EU) 등 현지화 필요

**행동(결제) 단계 변화**
- 결제: 토스페이먼츠 → Stripe (글로벌 표준)
- 가격: $9.99 (one-time), $14.99/month (Pro) — PPP 고려 필요
